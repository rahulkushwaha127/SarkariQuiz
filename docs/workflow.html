Workflow of Current System

This document describes what is already implemented in the app (Admin ¬∑ Creator ¬∑ Student ¬∑ Contests ¬∑ Clubs ¬∑ Notifications ¬∑ Practice ¬∑ Daily ¬∑ PYQ).

üß© Core Concepts

Roles

  • student main user who plays quizzes, joins contests, uses practice/revision, joins clubs
  • creator creates quizzes, generates questions with AI, creates contests, analytics, notifies students
  • super_admin moderates quizzes/contests, manages taxonomy, ads, settings, announcements
  • guest auto-created temp users for limited public play (not for clubs)

Routing

  • Public: homepage, browse exams/quizzes, daily, leaderboards, creator bio page
  • Student: /student/* inside mobile-first UI + sidebar
  • Creator: /creator/* modern Tailwind panel
  • Admin: /admin/* modern Tailwind panel
  • Post-login redirect: /dashboard routes to role dashboard

üîê Authentication

Login methods

  • Email + password (Laravel UI)
  • Optional Google login (Socialite) via /auth/google/redirect ‚Üí /auth/google/callback
Google login behavior:
  • Links user by google_id, else by email
  • If user does not exist: creates new user with student role
  • Blocked users (blocked_at) cannot log in

üåç Public (No Login)

  • Exam-focused homepage and browse pages
  • Public quizzes + public contests listing
  • Public leaderboards (daily/weekly/monthly/all-time + exam filter)
  • Daily challenge public page
  • Creator public bio page: /c/{username}

Guest play (no login)

  • Public quiz play auto-creates a temporary guest student user
  • Daily attempt limit is enforced

Student dashboard

  • Daily streak shown (current + best)
  • Quick access to daily, practice, revision, clubs, leaderboards

Quiz play (student)

Flow
  • Open a public/published quiz ‚Üí Start ‚Üí attempt created
  • Question-by-question answers
  • Auto submit when timer ends
  • Result screen: score + review + share card link
  • Bookmark/unbookmark questions (Revision)

  • System picks random questions from published + public quizzes
  • Result review + ‚ÄúRevise incorrect again‚Äù

Revision

  • Bookmarks from quiz/practice result screens
  • Mistakes list (incorrect answers)
  • ‚ÄúRevise again‚Äù flow reuses practice engine style

Daily challenge + streak

  • Admin selects daily quiz
  • When student submits that daily quiz ‚Üí streak tables update
  • Daily leaderboard exists

üßæ PYQ Bank (Previous Year Questions)

Student PYQ

  • Route: /student/pyq
  • Filter by exam/subject/topic/year
  • Start timed PYQ test (total time or per-question time)
  • Result + review screen

Admin PYQ

  • Route: /admin/pyq
  • Create/edit questions via AJAX modal (4 options + correct)
  • CSV import via modal (/admin/pyq/import)
  • Import logs stored for summary / error preview
Data: pyq_questions, pyq_answers, pyq_attempts, pyq_attempt_answers, pyq_import_logs
